Sieve of Eratosthenes: Finding Primes Efficiently

Understand the ancient yet highly efficient algorithm for finding all prime numbers up to a specified limit, with code illustrations.

KJ
Katherine Johnson
Published July 8, 2026
6 min read 1,420 views
Medium

1. The Search for Primes

If you need to find all prime numbers up to a given limit \( n \), the simplest method is to write a helper function that checks if each number \( x \) is prime using trial division (dividing by integers up to \( \sqrt{x} \)).

However, performing trial division on every number in the list results in a computational complexity of: \[ O(n \sqrt{n}) \] which is sluggish for larger limits. To scale efficiently, we require a sieve approach.

2. How the Sieve Works

Named after the ancient Greek scholar Eratosthenes of Cyrene, the algorithm operates by iteratively filtering out composite numbers.

Step-by-step Walkthrough:

  1. Create a list of consecutive integers from \( 2 \) through \( n \): \( (2, 3, 4, \dots, n) \).
  2. Initially, let \( p = 2 \), the smallest prime number.
  3. Enumerate the multiples of \( p \) by counting in increments of \( p \) from \( 2p \) to \( n \), and mark them in the list (these will be composite: \( 2p, 3p, 4p, \dots \)). Note that \( p \) itself is not marked.
  4. Find the first number greater than \( p \) in the list that is not marked. If there was no such number, stop. Otherwise, let \( p \) now equal this new number (which is the next prime), and repeat from step 3.
  5. When the algorithm terminates, the numbers remaining unmarked in the list are all the primes below \( n \).

As a smart optimization, we can start marking multiples of \( p \) directly from \( p^2 \) rather than \( 2p \), because all smaller multiples (\( 2p, 3p, \dots \)) will have already been marked by smaller prime passes. As a corollary, we only need to loop \( p \) up to \( \sqrt{n} \).

3. JavaScript Implementation

Here is the standard implementation of the Sieve of Eratosthenes in JavaScript: 

function sieveOfEratosthenes(n) {
    // Create boolean array initialized to true
    let isPrime = new Array(n + 1).fill(true);
    isPrime[0] = isPrime[1] = false;
    
    for (let p = 2; p * p <= n; p++) {
        // If isPrime[p] is not changed, then it is a prime
        if (isPrime[p] === true) {
            // Update all multiples of p starting from p^2
            for (let i = p * p; i <= n; i += p) {
                isPrime[i] = false;
            }
        }
    }
    
    // Collect all prime numbers
    let primes = [];
    for (let i = 2; i <= n; i++) {
        if (isPrime[i]) primes.push(i);
    }
    return primes;
}

4. Complexity Analysis

The Sieve of Eratosthenes has highly efficient computational characteristics:

Complexity Metrics:
  • Time Complexity: \( O(n \log \log n) \) operations, which is extremely close to linear.
  • Space Complexity: \( O(n) \) to store the boolean flag status of each index.

This logarithmic-logarithmic factor is a consequence of the prime harmonic series. The sum of the reciprocals of prime numbers up to \( n \) grows as \( \log \log n \), which is why the sieve performs so few checks compared to other sorting/filtering structures.
